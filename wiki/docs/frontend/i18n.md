# Internationalization (i18n)

The frontend supports English and Chinese (Simplified) using **react-i18next**. All user-facing text is externalized into JSON locale files, making it easy to add new languages.

## Setup

The i18n configuration is in `src/i18n/index.ts`:

```typescript
import i18n from 'i18next'
import { initReactI18next } from 'react-i18next'
import LanguageDetector from 'i18next-browser-languagedetector'

import en from './locales/en.json'
import zhCN from './locales/zh-CN.json'

i18n
  .use(LanguageDetector)
  .use(initReactI18next)
  .init({
    resources: {
      en: { translation: en },
      'zh-CN': { translation: zhCN },
    },
    fallbackLng: 'en',
    interpolation: {
      escapeValue: false, // React already escapes
    },
    detection: {
      order: ['localStorage', 'navigator'],
      caches: ['localStorage'],
    },
  })
```

### Key Configuration

| Setting | Value | Purpose |
|---|---|---|
| `fallbackLng` | `'en'` | Use English if a translation key is missing |
| `interpolation.escapeValue` | `false` | React handles XSS escaping |
| `detection.order` | `['localStorage', 'navigator']` | Check localStorage first, then browser language |
| `detection.caches` | `['localStorage']` | Save language preference to localStorage |

## Locale Files

Translations are stored in JSON files under `src/i18n/locales/`:

| File | Language |
|---|---|
| `en.json` | English (554 lines) |
| `zh-CN.json` | Chinese Simplified |

### Structure

Locale files use a hierarchical key structure:

```json
{
  "app": {
    "title": "IBKR Dashboard",
    "subtitle": "Portfolio Analytics",
    "version": "IBKR DASH v0.1.0",
    "techStack": "SQLite · FastAPI · React"
  },
  "nav": {
    "dashboard": "Dashboard",
    "positions": "Positions",
    "trades": "Trades",
    "cashFlows": "Cash Flows"
  },
  "dashboard": {
    "title": "Dashboard",
    "totalEquity": "Total Equity",
    "cash": "Cash",
    "loading": "Loading..."
  }
}
```

## Using Translations in Components

### Basic Usage

```tsx
import { useTranslation } from 'react-i18next'

function MyComponent() {
  const { t } = useTranslation()

  return (
    <div>
      <h1>{t('dashboard.title')}</h1>
      <p>{t('dashboard.loading')}</p>
    </div>
  )
}
```

### Interpolation

Use `{{variable}}` syntax for dynamic values:

```json
{
  "dashboard": {
    "ytdTwrHelper": "{{startDate}} to date"
  }
}
```

```tsx
t('dashboard.ytdTwrHelper', { startDate: '2024-01-01' })
// -> "2024-01-01 to date"
```

### Pluralization

Use `_one` and `_other` suffixes:

```json
{
  "dashboard": {
    "validPeriods": "{{count}} valid {{period}}"
  }
}
```

```tsx
t('dashboard.validPeriods', { count: 5, period: t('dashboard.tradingDays') })
// -> "5 valid trading days"
```

### Arrays

Some translations return arrays (e.g., welcome questions for the Copilot):

```json
{
  "copilot": {
    "welcomeQuestions": [
      "What is my total equity?",
      "What are my top 5 positions?",
      "What is my risk level?"
    ]
  }
}
```

```tsx
const questions: string[] = t('copilot.welcomeQuestions', { returnObjects: true })
```

## Language Switcher

The language toggle in `AppHeader.tsx` switches between English and Chinese:

```tsx
function toggleLanguage() {
  const next = i18n.language === 'zh-CN' ? 'en' : 'zh-CN'
  void i18n.changeLanguage(next)
}

<button onClick={toggleLanguage}>
  {i18n.language === 'zh-CN' ? 'EN' : '中文'}
</button>
```

The language preference is saved to `localStorage` and restored on page reload.

## How to Add New Translations

### 1. Add the key to both locale files

**en.json:**
```json
{
  "mySection": {
    "myKey": "My English text"
  }
}
```

**zh-CN.json:**
```json
{
  "mySection": {
    "myKey": "我的中文文本"
  }
}
```

### 2. Use the key in your component

```tsx
const { t } = useTranslation()
return <p>{t('mySection.myKey')}</p>
```

### 3. Fallback behavior

If a key is missing in the active language, i18next falls back to English (`fallbackLng: 'en'`). If the key is missing in English too, the key string itself is displayed.

## How to Add a New Language

1. Create a new locale file: `src/i18n/locales/ja.json` (Japanese example)
2. Copy the structure from `en.json` and translate all values
3. Register the locale in `src/i18n/index.ts`:

```typescript
import ja from './locales/ja.json'

i18n.init({
  resources: {
    en: { translation: en },
    'zh-CN': { translation: zhCN },
    'ja': { translation: ja },
  },
  // ...
})
```

4. Update the language switcher in `AppHeader.tsx` to include the new option
5. The `LanguageDetector` plugin will auto-detect the new language from the browser

## Translation Key Naming Conventions

- Use **camelCase** for keys: `totalEquity`, `failedToLoad`
- Group by **page/section**: `dashboard.title`, `positions.subtitle`
- Use **dot notation** for nesting: `admin.system.database`
- Keep values **sentence-case** for labels: "Total Equity"
- Keep values **UPPERCASE** for section headers in the UI: "KEY REASONS", "RISKS"
- Use `{{variable}}` for interpolation: `"Page {{page}} of {{totalPages}}"`
